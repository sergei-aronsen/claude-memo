If `origin` already exists pointing elsewhere, remove it with `git remote remove origin`.

**Step 2: Create the GitHub repository (per D-07, D-08)**
```bash
gh repo create sergei-aronsen/claude-memo --public --description "Persistent engineering memory for Claude Code — semantic search, automatic capture, Obsidian vault"
```

**Step 3: Add topics (per D-09)**
Topics cannot be set via `gh repo create` — use `gh repo edit` separately:
```bash
gh repo edit sergei-aronsen/claude-memo --add-topic claude-code --add-topic memory --add-topic obsidian --add-topic mcp
```

**Step 4: Add remote and push (per D-14, D-11)**
```bash
git remote add origin https://github.com/sergei-aronsen/claude-memo.git
git push -u origin main
```

**Step 5: Verify repo is accessible**
```bash
gh repo view sergei-aronsen/claude-memo --json name,description,visibility,repositoryTopics
```
Must show: public visibility, correct description, 4 topics.

If any step fails, report the error — do NOT retry blindly. Common issues:
- "already exists" on repo create → repo was created in a prior attempt, skip to Step 3
- "remote origin already exists" → run `git remote remove origin` then retry Step 4
- Push rejected → check if remote has commits (it shouldn't for a new repo)
