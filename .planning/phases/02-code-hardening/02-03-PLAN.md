  <action>
1. Install ruff: `pip3 install ruff==0.15.10` (or `uv pip install ruff==0.15.10`)

2. Run `ruff check scripts/` to see all violations. Expected categories per RESEARCH.md:
   - E722: bare `except:` (in hook scripts — already covered by per-file-ignores for save_raw_log.py and pre_compact_save.py)
   - F: unused imports (remove them)
   - W: pycodestyle warnings (fix whitespace issues)
   - I: import sorting (let ruff fix with `ruff check --fix --select I scripts/`)

3. Fix violations using this strategy (per D-05, pragmatic over dogmatic):
   - **Auto-fixable** (import sorting, trailing whitespace): Run `ruff check --fix scripts/`
   - **Unused imports (F401)**: Remove the unused imports manually if ruff --fix doesn't handle them
   - **Bare except (E722)**: If found in files OTHER than save_raw_log.py and pre_compact_save.py (which have per-file-ignores), change `except:` to `except Exception:`. Do NOT add more per-file-ignores — fix the code.
   - **Long lines (E501)**: Should not trigger since line-length is 120 per D-06. If any lines exceed 120, break them.
   - **Other violations**: Fix the code, don't suppress. Only use `# noqa: EXXX` as last resort with a comment explaining why.

4. Run `ruff check scripts/` again — must exit 0 with zero errors.

5. Also run `ruff format --check scripts/` to verify formatting. If formatting issues exist, run `ruff format scripts/` to fix them.

Do NOT modify pyproject.toml ruff config to add global ignores. Per RESEARCH.md anti-patterns: wholesale rule disabling hides real bugs.
  </action>

    - scripts/memo_engine.py (primary mypy target — core logic, argparse CLI, SQLite operations)
    - scripts/auto_memo.py (secondary mypy target — hook script with inline imports from memo_utils)
    - .planning/phases/02-code-hardening/02-RESEARCH.md (Pitfall 3: str = None pattern, Pitfall 4: inline imports)
  </read_first>
  <action>
1. Install mypy: `pip3 install mypy==1.20.1` (or `uv pip install mypy==1.20.1`)

2. Run `mypy scripts/memo_engine.py scripts/auto_memo.py` to see all errors. Per D-07 this is basic checking, not strict mode. Per D-08, `ignore_missing_imports = true` in pyproject.toml handles sentence-transformers, mcp, numpy stubs.

3. Expected error categories from RESEARCH.md:
   - **`fcntl` import** (Pitfall 1): If mypy complains about `fcntl` despite ignore_missing_imports, add `# type: ignore[import-not-found]` at the import line in memo_engine.py with comment `# Unix-only module`
   - **`str = None` defaults** (Pitfall 3): If these appear in memo_engine.py or auto_memo.py function signatures, fix to `str | None = None`. Example: `def call_haiku(prompt: str, system: str = None)` becomes `def call_haiku(prompt: str, system: str | None = None)`
   - **Inline imports** (Pitfall 4): auto_memo.py imports memo_utils inside functions. The `mypy_path = "scripts"` in pyproject.toml should resolve this. If mypy still can't find memo_utils, add `# type: ignore[import-not-found]` at those import sites.
   - **Untyped function definitions**: For basic checking, these show as warnings only if `disallow_untyped_defs = true` (which we are NOT setting per D-07). Should not appear.
   - **Other type errors**: Fix the actual type issues. Common fixes: add Optional/None to union types, add type annotations to variables mypy can't infer.

4. Fix errors directly in the source code. Prefer fixing types over adding `# type: ignore`. Use `# type: ignore[error-code]` (with specific error code, not bare `# type: ignore`) only when the fix would require restructuring code beyond this phase's scope.

5. Run `mypy scripts/memo_engine.py scripts/auto_memo.py` again — must exit 0.

Do NOT change mypy to strict mode. Do NOT expand mypy targets beyond memo_engine.py and auto_memo.py per D-07.
